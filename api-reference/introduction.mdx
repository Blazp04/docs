## API Overview

The SkyGuard API provides:

- **User Management**: Create, update, and manage user accounts
- **Authentication**: JWT-based authentication with refresh token rotation
- **Health Monitoring**: System health and status endpoints
- **Security**: Role-based access control and secure endpoints

## Base URL

When running locally, the API is available at:

```
http://localhost:3001/api/v1
```

## Authentication

All protected API endpoints require authentication using Bearer tokens. Include your JWT token in the Authorization header:

```bash
Authorization: Bearer <your-jwt-token>
```

Authentication tokens are obtained through the login endpoint and include:
- **Access Token**: Short-lived token for API requests (15 minutes)
- **Refresh Token**: Long-lived token for obtaining new access tokens (7 days)

## Response Format

All API responses follow a consistent format:

```json
{
  "success": boolean,
  "data": object | array,
  "message": string,
  "meta": {
    "timestamp": "ISO 8601 datetime",
    "version": "API version"
  }
}
```

## Error Handling

Error responses include detailed information to help with debugging:

```json
{
  "success": false,
  "message": "Error description",
  "error": {
    "code": "ERROR_CODE",
    "details": "Additional error details"
  },
  "meta": {
    "timestamp": "ISO 8601 datetime",
    "version": "API version"
  }
}
```
